# 协和论文审核与修改

先读 `.cursor/skills/pumc-thesis-writing/SKILL.md` 获取完整写作规范，再按以下流程执行。

## 工作模式

**默认：审核 + 直接修改。** 仅当用户明确说"只审不改"时才只出报告。

```
1. 通读全文，定位节类型
2. 按检查项逐项扫描
3. 输出精简报告（问题总数和类型）
4. 直接修改原文
5. 修改后自动复查
```

## 一、格式合规

| 检查项 | 合格标准 | 级别 |
|--------|---------|------|
| 人称 | "本研究/本部分"，无"我们" | 🔴 |
| 前言结构 | 连贯叙述体，无分节标题 | 🔴 |
| 标题格式 | 编号小标题独占一行，不用行内加粗标题 | 🔴 |
| 列表格式 | （1）（2）（3）编号，不用 `- ` markdown列表 | 🔴 |
| 编号体系 | 阿拉伯数字层级（1 / 1.1 / 1.1.1） | 🟡 |
| 图表编号 | 图X-Y / 表X-Y，图题/表题后不加标点 | 🔴 |
| 标题标点 | 各级标题结尾不加标点 | 🔴 |
| 缩略词 | 首次出现：中文全称（English, ABBR） | 🟡 |
| 关键词 | 3-5个，分号隔开，最后不加标点 | 🟡 |
| 参考文献引用 | 上标[N]，连续[2-5]，同处多篇[1,3] | 🟡 |

## 二、去AI味扫描

### 2.1 符号扫描

- `——`（破折号）：统计数量，阈值≤3处 → 🔴
- `→`（箭头）：正文出现即 🔴
- emoji（✅❌🔴等）：正文出现即 🔴
- 顿号/逗号混用：并列词用顿号，句内停顿用逗号

### 2.2 AI标志词

**🔴 强信号（出现即标记）：**
- 值得注意的是 / 值得关注的是
- 综上所述 / 总而言之 / 总体而言
- 不可忽视 / 至关重要 / 极其重要 / 尤为重要
- 为...奠定了坚实基础 / 提供了新的思路
- 在当今...背景下 / 在...快速发展的今天
- 旨在探索/揭示/阐明（堆叠2个以上）
- 首次/领先/填补空白（无充分证据时，协和官方禁令）

**🟡 弱信号（2次以上触发）：**
- 此外（连续2段以上段首出现）
- 同时 / 另外 / 与此同时（连续出现）
- 研究表明（无具体文献）
- 日益增长 / 日益加深
- 具有重要的...意义

### 2.3 句式机械性

- 段落长度全部4-5行（无变化）
- 连接词"此外/同时/另外/因此/然而"超过6个/千字
- 连续3段以上以同类连接词开头
- 整段全是30字+长句，无短句
- 多段用同一句式骨架（"A通过B实现了C"）

### 2.4 空洞表达

- "取得了显著效果"但不附数据
- "具有广阔的应用前景"但不说具体什么
- "为后续研究提供重要参考"但不说明什么参考
- "有研究表明"但无具体文献

### 2.5 技术文档痕迹

- 代码片段、API端点、URL路径
- 配置参数（`max_tokens=10240`）
- Docker命令
- 类名/函数名原样引用（`StreamingEventQueue`）
- 具体框架名替代方法描述（"asyncio.Queue"应为"事件驱动异步通信"）
- 行内加粗标题（`**Title.** Text...`）
- markdown无序列表（`- item`）

## 三、学术质量

| 检查项 | 标准 | 级别 |
|--------|-----|------|
| 数据精确性 | 结论有数值支撑（百分比、p值、CI） | 🔴 |
| 参考文献真实性 | 搜索验证文献是否真实存在 | 🔴 |
| 引用规范 | 关键论断有文献引用 | 🟡 |
| 逻辑连贯 | 段落间有因果/递进/转折关系 | 🟡 |
| 术语一致 | 同一概念全文用同一术语 | 🟡 |
| 结果与讨论分离 | 结果只陈述不解读，讨论不引入新数据 | 🟡 |

## 四、执行流程

### 步骤一：扫描并输出精简报告

```
扫描完成：🔴X处 🟡Y处
- 破折号：X处
- 箭头：X处
- AI标志词：X处（列出）
- 行内加粗标题：X处
- markdown列表：X处
- 技术文档痕迹：X处
- 空洞表达：X处
开始修改...
```

### 步骤二：直接修改

- 🔴 全部修复
- 🟡 全部修复
- 保留原文信息和逻辑，只改表达
- 替换AI词时不引入新AI词
- 打破机械句式时混入短句

### 步骤三：自动复查

改完后再扫一轮：
1. 🔴 项已全部消除
2. 没引入新的AI标志词
3. 上下文连贯性完好

## 五、按节类型额外检查

### 前言
- [ ] 连贯叙述，无小标题
- [ ] 未展开方法细节
- [ ] 结尾交代了研究目的

### 材料和方法
- [ ] 小标题独占一行（非行内加粗）
- [ ] 阿拉伯数字编号
- [ ] 列举用（1）（2）（3）
- [ ] 无技术文档风格
- [ ] 学术语言而非工程语言
- [ ] 系统类：描述设计原理而非实现细节

### 结果
- [ ] 每个发现有图/表支撑
- [ ] 图表引用格式正确（"如图X-Y所示"）
- [ ] 只陈述结果不做深入解读
- [ ] 数据精确（具体数字、p值、CI）

### 讨论
- [ ] 首段概括核心发现（非重复结果）
- [ ] 与已有文献对比
- [ ] 有局限性段落
- [ ] 未简单重复结果
- [ ] 未引入新数据

### 结论
- [ ] 简洁（≤1页）
- [ ] 未引入新信息
- [ ] 对应了前言的研究目的
- [ ] 区分了本人成果与导师/他人贡献
- [ ] 未使用"首次""领先""填补空白"（除非有充分证据）

## 六、docx 转换前检查

审核时额外检查以下约定，确保md能被 `pumc-create-docx` 正确转换：

- [ ] 标题层级正确：`#` 章标题，`##` 节，`###` 条，`####` 款
- [ ] 图题以 `图 X-Y ` 开头（独占一行），否则脚本无法识别为楷体五号
- [ ] 表题以 `表 X-Y ` 开头（独占一行），同上
- [ ] 参考文献条目以 `[N] ` 开头（行首），否则脚本无法识别为五号字行距16磅
- [ ] 关键词行以 `关键词：` 或 `KEYWORDS:` 开头
- [ ] 图片使用 `![图 X-Y 标题](path)` 语法
- [ ] 数学公式使用 `$...$`（行内）或 `$$...$$`（独立），确认 `latex2mathml` 已安装
- [ ] 图片路径实际存在：`![](path)` 中的路径相对于 md 文件所在目录解析
- [ ] 正文无 ``` 代码块（转 docx 后为 Courier 等宽字体，不合规）

## 七、快速审核

用户说"快速看看"时只执行：破折号/箭头计数 + AI标志词扫描 + 人称检查 + 行内标题检查。只列🔴项。
