# Markdown 转协和博士论文 DOCX

将 `.md` 论文稿转换为符合协和2025版官方格式规范的 `.docx` 文件。

## 前置条件

1. 输入的 `.md` 文件应遵循 `pumc-thesis-writing/SKILL.md` 第十三节"Markdown 写作约定"中的格式规范
2. 转换前建议先用 `pumc-thesis-review` 审核一遍
3. 需要 `python-docx`、`Pillow`（用于图片尺寸检测）和 `latex2mathml`（用于数学公式转换，可选）

## 使用方法

```bash
python .cursor/skills/pumc-create-docx/scripts/md2docx.py input.md output.docx
```

如果未指定 output，默认输出为 `input.docx`（与输入同名）。

## 转换后 Agent 应自动执行的检查

转换完成后，Agent 应主动验证：

1. **文件大小**：如果 md 中有图片引用，docx 应明显大于纯文本（通常 >1MB）。如果很小说明图片没插进去
2. **图片完整性**：检查是否所有图片都成功插入（脚本不会报告成功/失败，需要通过文件大小间接判断）
3. **图片路径**：md 中的相对路径基于 md 文件所在目录解析。如果图片在其他目录，需要先复制/软链到正确位置

## 协和2025版格式规范速查

以下为脚本中实现的全部格式参数，来源于《北京协和医学院研究生学位论文写作规范》（2025年7月版）。

### 页面设置
- A4纸（210mm x 297mm）
- 边距：上 2.5cm，下 2.5cm，左 3.0cm，右 3.0cm
- 页眉页脚距边界 1.5cm

### 正文标题层级

| 层级 | 示例 | 中文字体 | 英文/数字字体 | 字号 | 对齐 | 段前 | 段后 | 行距 |
|------|------|---------|-------------|------|------|------|------|------|
| 章标题 | 第 1 章 绪论 | 黑体 | Times New Roman | 三号(16pt) | 居中 | 24磅 | 18磅 | 单倍 |
| 二级 | 1.1 标题 | 黑体 | Times New Roman | 小三(15pt) | 顶左无缩进 | 24磅 | 6磅 | 单倍 |
| 三级 | 1.1.1 标题 | 黑体 | Times New Roman | 四号(14pt) | 顶左无缩进 | 12磅 | 6磅 | 单倍 |
| 四级 | 1.1.1.1 标题 | 宋体 | Times New Roman | 小四(12pt) | 左缩进2字符 | 12磅 | 6磅 | 单倍 |

> 注意：只有四级标题缩进2字符，一至三级均顶格左对齐。

### 正文段落
- 中文：宋体小四号(12pt)
- 英文/数字：Times New Roman 小四号(12pt)
- 两端对齐，首行缩进2字符
- 段前段后 0磅，行距 20磅

### 摘要
- 标题"摘要"：黑体三号(16pt)，居中，段前24磅，段后18磅
- 正文：宋体小四号(12pt)，首行缩进2字符，行距20磅
- "关键词："：黑体小四号(12pt)，内容宋体小四号

### 图表题
- 图序图题：楷体五号(10.5pt)，居中，编号与题名间空1字符，题后不加标点
- 表序表题：楷体五号(10.5pt)，居中，同上
- 表内容：宋体五号(10.5pt)

### 参考文献
- 标题：黑体三号(16pt)，居中，段前24磅，段后18磅
- 内容：宋体五号(10.5pt)，英文Times New Roman，行距16磅

### 页眉页码
- 页眉：宋体小五号(9pt)，英文Times New Roman，居中
- 页码：Times New Roman 小五号(9pt)，居中

## Markdown 标题映射规则

脚本按以下规则将 markdown 标题映射为 Word 样式：

| Markdown | 映射为 | Word样式 |
|----------|--------|---------|
| `# 标题` | 章标题 | 黑体三号居中 |
| `## 标题` | 二级标题 | 黑体小三号左对齐 |
| `### 标题` | 三级标题 | 黑体四号左对齐 |
| `#### 标题` | 四级标题 | 宋体小四号缩进 |
| 正文段落 | 正文 | 宋体小四号缩进 |

## 字体要求

脚本会对每个段落中的文本逐字符检查：
- 中文字符：使用宋体（标题用黑体，图表题用楷体）
- 英文字母和阿拉伯数字：统一使用 Times New Roman
- 这是协和官方规范的硬性要求

## 图片处理（重要）

### 自动嵌入机制

markdown 中 `![图1-1 标题](path/to/image.png)` 会：

1. **优先插入实际图片**：基于 md 文件所在目录解析相对路径，找到图片文件后以嵌入式（inline）插入
2. **自动适配尺寸**：使用 PIL 读取图片原始宽高比，约束最大宽度 14cm、最大高度 20cm，等比缩放
3. **独立段落格式**：图片段落使用居中对齐、无首行缩进、单倍行距、前后留白，不套用正文样式（避免固定行距压扁图片）
4. **回退机制**：若图片文件不存在，生成 `[此处插入图片]` 占位符
5. **图题输出**：`![]()` 方括号内的文本作为图题，以楷体五号居中输出在图片下方

### 图片路径注意事项

- 路径相对于 md 文件所在目录解析（不是工作目录）
- 如果原始图片在其他目录（如 `Case/Figure 2.png`），需要先复制或软链到 md 引用的路径（如 `figures/fig3-2.png`）
- Agent 在转换前应确认所有引用的图片文件存在

### 图注（Legend）最佳实践

图注直接写在 `![]()` 的方括号内，格式为：

```
![图 3-2 主标题。A. 子图A说明。B. 子图B说明。...](figures/fig3-2.png)
```

脚本会将方括号内的完整文本作为图题输出（楷体五号居中）。图注应包含：
- 主标题（概括全图内容）
- 每个子图（A/B/C/D...）的简要说明
- 显著性标注（如 *P < 0.05, **P < 0.01 等）

## 其他注意事项

- 脚本处理内容格式，不生成封面/首页/独创性声明（这些用学校模板）
- 参考文献建议用 EndNote/Zotero 管理，脚本只处理基础格式
- **正文中 `[1]`、`[2-5]` 等引用标记自动转为上标**，文后参考文献列表的 `[1]` 编号保持正常大小
- **第一个 `#` 标题不会在前面插入分页符**（避免空白首页），后续 `#` 标题自动分页
- 生成的 docx 可直接粘贴到学校官方 Word 模板中
- 依赖：`pip install python-docx Pillow latex2mathml`（latex2mathml 为可选依赖，未安装时公式回退为纯文本）

## 数学公式处理

### 支持的语法

| 语法 | 用途 | Word 渲染效果 |
|------|------|-------------|
| `$\alpha = 0.5$` | 行内公式 | 嵌入在段落文本中的可编辑公式对象 |
| `$$s = \alpha \cdot x + (1-\alpha) \cdot y$$` | 独立公式（单行） | 居中的独立公式段落 |
| `$$`...多行...`$$` | 独立公式（多行） | 合并为单行后居中输出 |

### 支持的 LaTeX 构造

下标 `x_{i}`、上标 `x^{2}`、分数 `\frac{a}{b}`、根号 `\sqrt{x}`、希腊字母 `\alpha`/`\beta`/`\gamma`、运算符 `\cdot`/`\pm`/`\times`、集合 `\in`/`\{...\}`、括号自适应。覆盖协和论文中常见的数学表达。

### 工作原理

1. LaTeX 经 `latex2mathml` 转为 MathML
2. 内置 MathML-to-OMML 转换器将 MathML 转为 Word 原生的 Office Math Markup Language（OMML）
3. OMML XML 直接插入 docx 段落，Word 识别为可编辑公式对象

### 降级机制

- 若 `latex2mathml` 未安装：所有 `$...$` 和 `$$...$$` 原样输出为纯文本
- 若某个公式转换失败：该公式回退为纯文本，不影响其他公式和正文
